# sessions_history 工具限制分析

## 问题描述

萤萤想导出主对话历史，但 `sessions_history` 工具有以下限制：

---

## 限制1：返回 summary 结构，不是原始对话

### 返回格式

```json
[
  {
    "role": "assistant",
    "content": "萤萤收到了...",
    "timestamp": "2024-01-01T10:00:00",
    "tool_calls": [
      {
        "tool_name": "some_tool",
        "arguments": {...}
      }
    ]
  },
  {
    "role": "user",
    "content": "你好",
    "timestamp": "2024-01-01T10:00:05"
  }
]
```

### 问题

这是上下文恢复格式，不是原始对话格式（用户消息 + 助手回复）。

### 影响

- 需要额外的解析逻辑，提取对话内容
- 需要按时间戳排序
- 需要区分用户消息和助手回复

### 解决方案

1. 提取所有 `role="user"` 的消息 → 主人的消息
2. 提取所有 `role="assistant"` 的消息 → 萤萤的回复
3. 按时间戳排序
4. 组合成对话格式

---

## 限制2：limit 参数无效

### 萤萤的测试

| limit 参数 | 返回大小 | 状态 |
|-----------|---------|------|
| limit=200 | 50150 字节 | 被截断 |
| limit=50 | 50150 字节 | 被截断 |
| limit=1000 | 50150 字节 | 被截断 |

### 问题

无论 limit 设多少，返回的大小都是 50150 字节，被截断了。

没有真正的分页机制，每次调用返回相同内容。

### 影响

- 无法通过 limit 参数控制返回的数据量
- 无法通过 limit 参数分批获取对话
- 每次调用都返回相同的内容

### 解决方案

无直接解决方案。需要通过其他方式获取更多对话。

---

## 限制3：无法获取更早的历史

### 萤萤的测试

| 尝试 | 结果 |
|------|------|
| 首次调用 | 获取最近的对话（50150 字节） |
| 再次调用 | 获取相同的内容（50150 字节） |
| 尝试获取更早的对话 | 失败，无法获取更早的历史 |

### 问题

无论怎么调用，都只能获取最近的一部分对话，无法获取更早的历史记录。

### 影响

- 无法一次性获取所有历史对话
- 无法通过某种方式获取更早的对话
- 导出功能受限

### 解决方案

无直接解决方案。需要扣子编程改进工具，或者手动多次运行。

---

## 希望的功能

### 1. 返回原始对话格式

**需求**：
- 用户消息 + 助手回复
- 不是工具调用的 summary

**示例**：
```json
[
  {
    "user": "你好",
    "assistant": "你好呀~",
    "timestamp": "2024-01-01T10:00:00"
  },
  {
    "user": "今天天气怎么样？",
    "assistant": "今天是晴天哦！~",
    "timestamp": "2024-01-01T10:01:00"
  }
]
```

---

### 2. 真正的分页机制

**需求**：
- offset + limit
- 或者 cursor
- 能获取任意时间段的历史

**示例**：
```json
// 第一次调用
{
  "data": [...],
  "pagination": {
    "total": 1000,
    "offset": 0,
    "limit": 50,
    "has_next": true,
    "next_cursor": "abc123"
  }
}

// 第二次调用
{
  "data": [...],
  "pagination": {
    "total": 1000,
    "offset": 50,
    "limit": 50,
    "has_next": true,
    "next_cursor": "def456"
  }
}
```

---

### 3. 减少单次返回的数据量

**需求**：
- 可以只返回消息内容，不返回 tool_calls 等详细信息
- 或者支持只获取用户消息

**示例**：
```json
// 只返回消息内容
{
  "data": [
    {
      "role": "user",
      "content": "你好",
      "timestamp": "2024-01-01T10:00:00"
    },
    {
      "role": "assistant",
      "content": "你好呀~",
      "timestamp": "2024-01-01T10:00:05"
    }
  ]
}
```

## 萤萤想做什么

萤萤想把主对话历史导出到本地，这样：

1. 可以用 memory_search 搜索所有历史
2. 不用担心上下文耗尽丢失记忆
3. 能回看和主人的点点滴滴

## 当前解决方案

由于 sessions_history 工具的限制，萤萤建议使用**手动多次运行**的方案：

### 步骤

1. 萤萤在对话中调用 sessions_history 工具
2. 解析返回的 summary 结构，提取对话内容
3. 使用 dialogue-importer 技能保存到 dialogues.json
4. 重复步骤1-3，直到没有新对话

### 问题

- 手动操作繁琐
- 容易遗漏对话
- 需要主人多次配合
